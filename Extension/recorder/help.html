liveDownload Help Guide

What is liveDownload?

liveDownload is a Chrome extension for unattended live stream recording and VOD downloading from any HLS/DASH-based streaming platform. Set it up, walk away, and come back to complete recordings.

Quick Start

Manual recording

  1. Navigate to a live stream page (e.g. play.livebroadcast.com/broadcaster)
  2. Click the liveDownload icon — the recorder window opens
  3. Detected streams appear under Available Streams
  4. Select a stream and click Download Selected

Auto-recording (unattended)

  1. Click the liveDownload icon to open the recorder window
  2. Expand the Wait for Recording URLs section
  3. Paste broadcaster URLs and click + Add
  4. liveDownload polls each URL every 15 minutes. When a stream goes live, recording starts automatically.

Setting a Recording Folder

By default, recordings go to your browser's Downloads folder. To save to a specific folder:

  1. Click ⚙️ Settings in the header
  2. Under Root Download Directory, click Choose Folder
  3. Select your folder and click Save Settings
Tip: The folder selection is supposed to survives browser restarts. However, Chrome seems to behave differently at different times. The best way to ensure that your recording always goes to the selected folder is change the Download folder in browser setting. Another approach that seems to work: After selecting the folder of choice, start a manual recording and grant permission using the button on the top right corner of the window.

Wait for Recording URLs (WRU)

The WRU system is the heart of unattended recording. Add broadcaster URLs and liveDownload polls them automatically.

Recovery Mode

If the stream is interrupted (authentication expiry, broadcaster restart, network failure), liveDownload enters Recovery Mode:

  • The current file is finalized and saved
  • A background tab opens the broadcaster's page
  • When the stream reappears, recording resumes into a new file automatically
  • Recovery times out after 10 minutes if no stream is found

Stopping

Click ⬛ STOP RECORDING to finalize and save. The file is written immediately — no post-processing needed.

VOD Downloads

For pre-recorded content, liveDownload detects m3u8/mpd playlist URLs on the page and lists them under Available Streams.

  • Single download — select one stream and click Download Selected
  • Batch download — select multiple streams, click Download Selected, choose a folder — each stream saves as a separate file

Output Files

Live recordings are saved as .ts (MPEG-TS) files by default.

Filename format: [Title]-[Mon]-[DD]-[YYYY]-[H]-[MM]-[SS][AM/PM].ts

Example: BroadcasterName-Apr-04-2026-10-33-05PM.ts

Recovery files from the same session share the same base name with a new timestamp.

Converting to MP4

ffmpeg -i recording.ts -c copy output.mp4

Or use the included TS2MP4.PS1 PowerShell script for batch conversion with logging:

.\TS2MP4.PS1 -i "D:\Recordings" -o "D:\MP4"
Note: ffmpeg may show "Packet corrupt" warnings when converting some recordings. This is cosmetic — the DTS values from recording's session clock are high but valid. Output quality is unaffected.

Privacy

  • No telemetry. liveDownload does not collect, transmit, or store any usage data.
  • No ads. No advertising of any kind.
  • Google Translate is used only if you enable Auto-Translate Titles (off by default). When enabled, stream titles are sent to Google Translate.
  • Autoplay assistance. To start stream playback on monitored pages, liveDownload injects mouse click events into the player area.
  • Sound permission. liveDownload sets Chrome's sound permission to "Allow" for broadcaster origins you add to the Watch List, to ensure autoplay works reliably.

Troubleshooting

No streams detected

Try enabling Detect Streams in XHR Requests in Settings. Some players load streams via JavaScript and won't appear in the default network log.

Recording saves to Downloads instead of my folder

Make sure you click Save Settings after choosing a folder — not just choose it. After saving, a file called liveFolder.TXT should appear in your chosen folder. If it doesn't, the folder was not saved correctly.

Auto-record didn't start

Check that the URL in WRU matches the broadcaster's base URL exactly (e.g. play.livebroadcast.com/username). Use the 🔄 Poll Now button to trigger an immediate check rather than waiting for the next scheduled poll.

ffmpeg shows "Packet corrupt" warnings

This is expected for some recordings and is cosmetic. The output file plays correctly.
